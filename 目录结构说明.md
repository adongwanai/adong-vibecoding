# Everything Claude Code - 目录结构详解

## 📁 核心配置目录

### 🤖 agents/ - 专业代理
**作用：** 包含各种专门的 AI 代理，用于处理特定任务

**主要文件：**
- `planner.md` - 功能规划代理，制定详细实施计划
- `architect.md` - 系统架构设计，技术决策
- `tdd-guide.md` - TDD 测试驱动开发强制执行
- `code-reviewer.md` - 代码质量审查
- `security-reviewer.md` - 安全漏洞扫描
- `build-error-resolver.md` - 构建错误修复
- `e2e-runner.md` - E2E 端到端测试
- `refactor-cleaner.md` - 死代码清理
- `doc-updater.md` - 文档同步更��
- `tri-ai-orchestrator.md` - **你新增的 Tri-AI 工作流协调器**

**使用方式：** Claude Code 会根据任务自动委托给合适的代理

---

### 🎯 commands/ - 斜杠命令
**作用：** 快捷命令，用 `/command-name` 直接调用

**主要命令：**
- `plan.md` → `/plan` - 规划新功能
- `tdd.md` → `/tdd` - 测试驱动开发
- `code-review.md` → `/code-review` - 代��审查
- `e2e.md` → `/e2e` - E2E 测试
- `build-fix.md` → `/build-fix` - 修复构建错误
- `refactor-clean.md` → `/refactor-clean` - 清理死代码
- `triai-frontend.md` → `/triai-frontend` - **你新增的前端工作流**
- `triai-backend.md` → `/triai-backend` - **你新增的后端工作流**
- `triai-debug.md` → `/triai-debug` - **你新增的调试工作流**

**使用方式：** 在 CLI 中直接输入 `/plan` 等命令

---

### 📚 skills/ - 领域知识库
**作用：** 工作流定义和最佳实践知识

**主要内容：**
- `coding-standards/` - 编码规范（TypeScript, JavaScript 等）
- `backend-patterns/` - 后端模式（API、数据库、缓存）
- `frontend-patterns/` - 前端模式（React, Next.js）
- `tdd-workflow/` - TDD 工作流
- `security-review/` - 安全检查清单
- `continuous-learning/` - 自动提取会话模式
- `strategic-compact/` - 上下文压缩策略

**使用方式：** 被 agents 和 commands 自动调用

---

### 📏 rules/ - 强制规则
**作用：** 必须遵守的开发准则（复制到 `~/.claude/rules/`）

**主要规则：**
- `security.md` - 安全检查（禁止硬编码密钥等）
- `coding-style.md` - 代码风格（不可变性、文件组织）
- `testing.md` - 测试要求（TDD、80% 覆盖率）
- `git-workflow.md` - Git 工作流（提交格式、PR 流程）
- `agents.md` - 代理使用规则
- `performance.md` - 性能优化（模型选择、上下文管理）

**使用方式：** Claude Code 始终遵守这些规则

---

### 🪝 hooks/ - 自动化钩子
**作用：** 基于事件的自动化触发器

**主要功能：**
- `hooks.json` - 钩子配置（PreToolUse, PostToolUse, Stop 等）
- `memory-persistence/` - 会话记忆持久化
- `strategic-compact/` - 上下文压缩建议

**使用方式：** 在特定事件发生时自动执行

---

## 🔧 工具和脚本目录

### 🛠️ scripts/ - 跨平台脚本
**作用：** Node.js 脚本，支持 Windows/macOS/Linux

**主要脚本：**
- `lib/utils.js` - 跨平台工具函数
- `lib/package-manager.js` - 包管理器检测
- `hooks/session-start.js` - 会话开始时加载上下文
- `hooks/session-end.js` - 会话结束时保存状态
- `hooks/evaluate-session.js` - 从会话提取模式
- `setup-package-manager.js` - 交互式包管理器设置

---

### 🧪 tests/ - 测试套件
**作用：** 脚本和钩子的单元测试

**主要内容：**
- `lib/` - 库测试
- `hooks/` - 钩子测试
- `run-all.js` - 运行所有测试

**使用方式：** `node tests/run-all.js`

---

## 💡 核心理解

**这个配置库的本质：**

```
agents/    →  大脑们（专业 AI 代理）
commands/  →  快捷键（斜杠命令）
skills/    →  知识库（最佳实践）
rules/     →  规则书（强制遵守）
hooks/     →  自动化（事件触发）
```

**你的 Tri-AI 扩展：**

```
agents/tri-ai-orchestrator.md  → 总协调器
commands/triai-*.md            → 你的工作流命令
TRIAI-QUICKSTART.md            → 中文快速指南
examples/triai-*.md            → 实战案例
```
